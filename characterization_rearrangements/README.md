# Genomic Analysis: Conserved vs. Rearranged Regions 

This repository contains the R-based analysis pipeline for comparing conserved (collinear) and rearranged genomic regions, alongside statistical breakpoint enrichment analysis.

---

## 📂 Repository Contents

### 1. Characterization & Visualization
*   **`window_analisis.R`**: Performs Generalized Linear Models (GLM) to characterize genomic regions.
*   **`violin_graph.R`**: Generates violin plots for visualizing feature distributions (edited via InkScape for publication).
*   **`genome_features_1JMC18.R` / `genome_features_4JMC19C.R`**: Generates Supplementary Figures S2 and S3, visualizing genomic features along chromosomes (all vs. rearranged).

### 2. Breakpoint Enrichment Analysis
*   **`enrichment_analysis.R` / `enrichment_analysis_chr.R`**: Performs permutation-based enrichment analysis of genomic features across the genome and specific chromosomes using `regioneR`.
*   **`enrichment_analysis_chr_area_depend.R`**: Runs section-specific (subtelomeric vs. interstitial) breakpoint enrichment analyses.

### 3. Data Utilities
*   **`transform_robj_table_chr.R` / `transform_robj_table_chr_area_depend.R`**: Data tidying scripts to convert R objects into structured tables for statistical modeling and visualization.

---

## 📥 Required Inputs
To run these scripts, ensure the following files are available:

*   **Assemblies:** Clean genome assemblies (.fasta) for both genomes.
*   **GC Content:** TSV file with GC content per 100 kb window (.gc.tsv).
*   **Synteny:** Synteny hits output from GENESPACE (.synHits.txt).
*   **GFF/GFF3 Files:**
    *   **Annotation:** Generated by **Braker** for both genomes (.gff3).
    *   **Repeats:** RepeatMasker output (.fasta.out.gff).
    *   **RepeatExplorer2:** DANTE, DANTE_LTR, and TAREAN output files (.gff3).

---

## 🔬 Methodology Overview

### Breakpoint Enrichment
*   **Statistical Validation:** Enrichment of genomic features at breakpoints is evaluated using **1000 permutations** via the `regioneR` package.
*   **Statistics:** `numOverlaps` is used for categorical features; `meanInRegions` is used for GC content analysis.
*   **Spatial Controls:** Telomeric regions (terminal 10kb) are masked. Chromosomes are partitioned into subtelomeric (terminal 10%) and interstitial sections based on GC-content stabilization to ensure contextual accuracy.

## 💡 Usage Notes
Each R script is designed for independent execution. However, for a standard workflow, it is recommended to run `window_analisis.R` or `enrichment_analysis.R` first, as both generate the core tables required for downstream visualization and statistical modeling.
